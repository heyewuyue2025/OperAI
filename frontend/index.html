工作流地图

从素材到方案，
每一步都有依据。

Archive OS 把运营工作拆成五个清晰动作。它保留 AI 的生成能力，但让结果能复盘、能解释、能交付。

01任务输入

录入活动素材、目标平台和表达口径，明确生成边界。

02智能体编排

十个运营 Agent 作为可扩展岗位，默认链路先跑 D/C/N。

03结果复盘

每次执行都记录步骤、状态、耗时、输出和降级标记。

04证据核查

原始素材、指标、洞察、文案、排期与校验串成可追溯链路。

05交付导出

最终 Markdown/Word 是可继续编辑、复核和提交的运营方案。

证据核查

生成不是终点，

D/C/N 每一步都写入 SQLite 与 JSONL。工作台会把这些记录呈现为可审阅的运行结果。

01 原始素材 -> 指标 / 证据摘录已捕获

03 C 草案 -> N 排期 / 平台规则已应用
